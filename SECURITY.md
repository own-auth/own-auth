# Security Policy

Own Auth handles authentication, sessions, API keys, and account recovery flows. Treat suspected security issues as private until they are fixed.

## Reporting a Vulnerability

Do not open a public issue for vulnerabilities.

Use GitHub private vulnerability reporting for this repository:

https://github.com/tina-li-xx/own-auth/security/advisories/new

Include:

- affected version or commit
- affected auth flow
- clear reproduction steps
- impact and any known workaround

Do not include live secrets, raw tokens, passwords, API keys, or private user data in the report.

## Supported Versions

Security fixes target the latest published version of `own-auth`.

## Disclosure

Security reports should stay private until a fix is available and a disclosure note is ready.
